`nix/secrets/k3s.yaml` is currently a plaintext placeholder. Before adding any real k3s cluster token or other secrets, replace it with actual `sops`-encrypted content.

Decision context:

- Initial Setup uses `sops-nix` for the k3s cluster token.
- Encrypted secret may be committed to git.
- Decrypted secret material must never be committed.
- Recipient model: one admin age key plus one host age identity per NixOS node, derived from each node's SSH host key.

Expected secret path/key:

```yaml
k3s:
  token: ENC[...]
```

Runtime path expected by NixOS modules:

```text
/run/secrets/k3s/token
```

Relevant files:

- `nix/secrets/k3s.yaml`
- `nix/modules/secrets.nix`
- `nix/modules/k3s-server.nix`
- `nix/modules/k3s-agent.nix`

Acceptance criteria:

- `nix/secrets/k3s.yaml` contains only sops-encrypted secret material.
- Real token is not present in git history after this point.
- `.sops.yaml` or equivalent recipient config exists.
- NixOS evaluation can find the `k3s/token` secret.
